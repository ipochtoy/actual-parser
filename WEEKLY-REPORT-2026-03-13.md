## 1. Общее состояние проекта

Chrome-расширение для парсинга заказов с **eBay**, **iHerb** и **Amazon** с экспортом в Google Sheets. Проект находится в рабочем состоянии.

### Поддерживаемые магазины

| Магазин | Статус | Ключевые особенности |
|---------|--------|---------------------|
| eBay | Работает | Order ID, трекинг, ISO-даты, smart scroll |
| iHerb | Работает | Order ID, трекинг, продукты |
| Amazon | Работает (с ограничениями) | 3 режима парсинга, TBA-треки, таймауты |

## 4. Известные проблемы и технический долг

### Amazon парсер
- **Множественные посылки в одном заказе** — `parseIndividualItemSimpleByTrackUrl` ищет только один товар для каждой Track-кнопки; остальные товары из той же посылки пропускаются (задокументировано в `CURRENT-STATUS.md`, планировалось исправить в v6.8.x)
- **Пагинация** — задокументированы проблемы с блокировкой Amazon (`PAGINATION-BLOCKED.md`)
- Наличие нескольких бэкап-версий файлов в репозитории (`content-amazon-v6.7.1.js`, `.STABLE-v6.7.4`, `.STABLE-v6.7.6`) — захламляет проект

### Общее
- Много документации/чейнджлогов разных версий (12+ md-файлов) — стоит консолидировать
- Бэкап-файлы (`.STABLE-*`, `-working.js`) хранятся в основной директории вместо git-веток
- `.gitignore` минимальный (47 байт) — возможно, не покрывает все нужные исключения
